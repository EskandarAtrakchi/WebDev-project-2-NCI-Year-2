Application’s Objectives

The idea is to provide the user with real-time data about the crypto market, as we know that the crypto market is volatile and the traders/investors who are interested in cryptocurrency prices and want to stay ahead of the market then they will need a website to check the crypto prices in real live-time.
The website provides the following:

  1. Fear and Greed index of the whole crypto market indicates if the majority of the traders are buying or selling, the index is calculated from 0 to 100. Zero means "Extreme Fear", while 100 means "Extreme Greed"
  2. The ID number retrieval for any cryptocurrency that exists in the crypto market, the API will retrieve the ID number of the cryptocurrency, and the reason for this is because the smaller the ID number of the cryptocurrency the more data about the prices of the currency that we have.
  3. The retrieval of the following a. Symbol of the currency b. ID c. Price (US-Dollar) d. 24h Volume (US-Dollar) e. Market Capitalization (US-Dollar) f. Percent Change (1h) g. Percent Change (24h) h. Percent Change (7d)
    This is extremely important if the user wants to know the volatile price of some specific currencies.

Development process

  1. backendAPI.js Functions

    A. Function 1: Fetching Data for Fear and Greed Index

    Description: Asynchronous function (loadFearAndGreedData) designed to fetch fear and greed index data from a specified API.

    How It Works:

    1. Uses the fetch function to request data from the settled URL.
    2. Parses the JSON response from the API.
    3. Dynamically generates HTML content to display fear and greed index information.

    Expected Results: Populates the designated HTML element (fearAndGreedDiv) with fear and greed index details.

     
    B. Function 2: Fetching Data by Listings API with Bar-Chart

    Description: Asynchronous function (loadCryptoData) responsible for fetching cryptocurrency data from a server-side and rendering it with a bar-chart.

    How It Works:

    1. Retrieves user input from the search bar.
    2. Fetches cryptocurrency data from the server.
    3. Filters and displays relevant data based on the user input.
    4. Dynamically creates a bar-chart using the Chart.js library here.

    Expected Results: Displays cryptocurrency information and generates a bar-chart based on user input.

     
    C. Function 3: Fetch, Load, and Display Ticker Data with Bar-Chart

    Description: Asynchronous function (loadTickerData) designed to fetch and display ticker data along with a bar chart.

    How It Works:

    1. Retrieves user input from the search bar.
    2. Fetches ticker data from the server.
    3. Processes and displays the data in a table format.
    4. Dynamically creates a bar-chart with logarithmic scaling using Chart.js.

    Expected Results: Presents ticker data in a table and generates a bar-chart based on user-specified symbols.

    Notes:

    • I am using logarithmic scaling in Chart.js because there are cryptocurrencies with very low prices such as 0.00001038$ SHIBA at the time of writing this document and at the same time the price of BTC is around 42,000$. SHIBA cannot be seen next to BTC is they were rendered together, and that's the use of logarithmic scaling
    • The development process prioritized asynchronous operations for smooth user interactions.
    • Data is dynamically rendered in HTML elements.
    • Error handling is implemented to address possible issues in fetching and processing data.
    • You can access all the functions here GitHub
    • A test plan was conducted on the three function for API retrieval and the excel sheet can be seen here Testing Plan

    Noting that all three methods are used to retrieve the data from the external server, and the JSON data is dynamically rendered to the webpage.

  3. Server.js

    a. Express Framework: The server is built using express.js, which simplifies route handling and HTTP creation.

    b. Express Middleware: express.json() parses the incoming JSON object from the external server, making the data accessible in the req.body object.

    c. CORS(): Enables Cross-Origin Resources Sharing, allowing the server to handle requests from different origins.

    d. Routes and External API Requests:

    1. Route 1: / (Index)
      Returns a JSON response with a "tested! Index is working." message to check if the server is running.
    2. Route 2: /ticker
      Asynchronously fetches data from the external API endpoint https://api.alternative.me/v1/ticker/ using Axios. Responds with the retrieved JSON data or returns an error message in case of an issue.
    3. Route 3: /listings
      Asynchronously fetches data from the external API endpoint https://api.alternative.me/v2/listings/ using Axios. Responds with the retrieved JSON data or returns an error message in case of an issue.
    4. Route 4: /documentation
      Similar to the listings route.

    Notes: The server is designed to handle asynchronous operations, ensuring non-blocking execution. Axios is used to simplify making HTTP requests to external APIs and handling responses. CORS middleware is essential for allowing cross-origin requests, and efforts are made to ensure the server provides a straightforward way to test its functionality and check if it's operational.

Technologies used

  1. Express.js

    A. Purpose: Framework for Node.js providing features to build the application.

    B. Usage: Used to handle routes for different endpoints ("/", "/ticker", "/listings", "/documentation"), enabling the server to respond with JSON data.

  2. Axios

    A. Purpose: According to Axios documentation, it’s useful for fetching data from an external webserver.

    B. Usage: Simplifies the process of making asynchronous HTTP requests to the external APIs:

    • https://api.alternative.me/v1/ticker/
    • https://api.alternative.me/v2/listings/

  3. CROS

    A. Purpose: Unauthorized access will be prevented.

    B. Usage: The job of CROS is to enable Cross-Origin HTTP requests from the client-side to the server. This is important to handle requests from different domains.

  4. Node.js

    A. Purpose: Allowing JavaScript server-side execution.

    B. Usage: The entire script is executed within the Node.js runtime and also handles HTTP requests.

  5. JSON

    A. Purpose: Lightweight data that is easy for humans to read.

    B. Usage: The server responds to the client with JSON data.

  6. Port Configuration

    A. Purpose: Defines the port number on which the server listens for incoming requests.

    B. Usage: The server listens on port 3000 (const port = 3000;).

  7. Express.json()

    A. Purpose: Middleware for parsing the incoming JSON requests, making the data accessible in the req.body object.

  8. Express.use(CROS())

    A. Purpose: Middleware for enabling cross-origin sharing, using this to allow the server to handle requests from the origin.

  9. Chart.js:

    Purpose: A JavaScript library for creating interactive and visually appealing charts.

    Usage: Included in the HTML file using the CDN link Chart.js .

    Benefits: Allows to create alot of types of charts, including bar charts, line charts, and more. It enhances the visualization of data on the client side. in this website a bar chart is used only.

Advantages and disadvantages of the chosen technology

Technology Advantages Disadvantages
HTML and DOM Manipulation - Facilitates dynamic updates to the content and structure of web pages. - May lead to complex and less maintainable code, especially in larger projects like this one.
JavaScript - Chart.js for visualization and node.js for server side execution. - defining routes can be difficult
Asynchronous Programming (Async/Await) - Allows handling asynchronous tasks without blocking the main thread. - Requires careful error handling and may lead to callback hell if not structured properly.
Fetch API - Provides a simple and modern interface for making asynchronous HTTP requests. - Lack of built-in support for handling HTTP errors can require additional error-checking code.
JSON (JavaScript Object Notation) - Lightweight data interchange format easy to read and write. - Limited support for more complex data types compared to some other serialization formats.
Chart.js - Easy-to-use library for creating interactive and visually appealing charts. - May not be suitable for extremely complex or specialized charting requirements.
Error Handling (try-catch) - Enhances code robustness by providing a mechanism to handle exceptions. - Overuse of try-catch blocks can make it harder to identify and fix underlying issues.
Event Handling (Event Listeners) - Enables response to user actions, enhancing interactivity. - Events can add complexity to the code.
Conditional Statements - control decision-making in the code. - Nested conditional statements can lead to code that is difficult to understand.
String Manipulation - Provides flexibility in constructing and manipulating strings. - string concatenation can lead to inefficient string handling.
ES6 Features - Introduces modern syntax and features for code improvements. - Compatibility issues with older browsers may require transpilation or polyfills.
Node.js - Single programming language (JavaScript) for both client and server-side, enhancing code consistency. - Single-threaded nature might lead to blocking in resource-intensive operations. Node.js may not be suitable for CPU-bound tasks.
Express.js - Lightweight and flexible web application framework, making it easy to build and scale applications. - May lack some features available in larger frameworks.
Axios - Promise-based, allowing asynchronous handling of HTTP requests. - Additional dependency, although minimal.
CORS (Cross-Origin Resource Sharing) - Enables cross-origin HTTP requests, which I think it is very important for web applications. - Misconfiguration can lead to security vulnerabilities.
Express.json() Middleware - Simplifies handling of incoming JSON requests. - Overhead if not needed for the specific application.
Express.use(cors()) Middleware - Enables cross-origin resource sharing for the Express application. - Misuse may lead to security vulnerabilities.
Port Configuration (Port 3000) - Defines the port for the server to listen on. - Port conflicts if another application is using the same port.

Application Features

  1. Node.js Usage

    The website is using Node.js for server-side JavaScript code execution.

  2. Express.js for Route Handling

    Express.js is used to handle the routes and to respond to different endpoints.

  3. External API Integration

    axios is used to make HTTP requests to external APIs:

    • a. https://api.alternative.me/v1/ticker/ for fetching ticker data.
    • b. https://api.alternative.me/v2/listings/ for retrieving cryptocurrency listings.

  4. Data Presentation

    The JSON format received from the external server is manipulated to present it to the user. Different routes are available for obtaining general information ("/"), ticker data ("/ticker"), cryptocurrency listings ("/listings"), and additional information ("/documentation").

  5. Fear and Greed Index Display

    There's a function (loadFearAndGreedData) dedicated to fetching and displaying data related to the fear and greed index. The data is presented in an HTML div.

  6. Cryptocurrency Listings and Bar Chart

    (loadCryptoData) fetches cryptocurrency listings from the external API ("/listings") and displays relevant information in an HTML div. Then a bar-chart is generated using Chart.js, displaying crypto IDs based on user input.

  7. Ticker Data and Bar Chart

    A third function (loadTickerData) retrieves ticker data from the external API ("/ticker") based on user input. The data is displayed in an HTML table, and a barchart is generated using Chart.js.

  8. User Input Handling

    The application handles user input, validating and processing it to retrieve relevant cryptocurrency data. Alerts are used to notify users of input errors or when no data is found for the entered symbols.

  9. Port Configuration

    The server listens on port 3000, as specified by the const port = 3000; configuration.

  10. Error Handling

    The application includes error handling mechanisms for situations where data cannot be fetched from external APIs or there are issues with the server.
